# 論理セル1/6化 計画

更新日: 2026-01-05

## 目的
- 論理セルを現在の1/6にし、グリッド線が1セルを表す状態に合わせる
- プレイヤー体感・部屋サイズ・当たり判定の見た目を維持する

## 方針
- 論理セル=1/6を`cellSize=1/6`として反映し、列/行/部屋寸法を6倍して世界サイズは維持する
- レンダリング負荷増大を抑えるため、ステージ生成は論理セルを6x6でまとめた描画用グリッドに変換して作成する
- グリッド線間隔は論理セルサイズに合わせる

## ステップ
- [ ] 現状のセル依存箇所を洗い出す（GridLayout参照、world<->cell変換、HUD表示、AI判定、ステージ生成）
- [ ] 論理グリッドを1/6化：cellSize=1/6、columns/rows/部屋寸法/corridor幅を6倍し、同一の部屋形状を維持
- [ ] ステージ描画の負荷対策：論理グリッドを6x6で縮約した描画グリッドを生成し、createStageFromGridに渡す（当たり判定は描画グリッドと整合）
- [ ] グリッド線/ミニマップ/セル表示を論理セル基準に合わせる
- [ ] 目視確認（部屋サイズ感、移動体感、壁当たり、ビット/NPCの行動、ミニマップ）
